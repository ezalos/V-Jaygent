Grade vocabulary — what every critic value means

Sources: taste.md (rubric), skills/vjay-iterate/SKILL.md §"YAML tail" (~line 702) and §shader-verdicts (~line 376). This glossary is the compressed truth; those files are the law.

Probe grade values

Every probe is a concrete question with a pass/fail criterion defined in taste.md. The value records both the judgment and the kind of evidence behind it:

ValueMeaningEvidence basis
passThe probe's criterion is met.Observed directly in the rendered frames/clips.
failThe criterion is not met.Observed directly in frames/clips.
weak / weak-passTechnically present but under-delivering — counts toward neither a clean pass nor a fail in spirit; flagged as improvement surface.Observed, but marginal.
shader-pass / shader-fail / shader-unclearThe probe could not be tested from headless captures (e.g. cursor reversibility needs multi-position samples the harness doesn't take). The critic instead reads the GLSL and judges whether the code structure implies pass/fail — and must cite the deciding shader line. shader-pass counts as a pass in the N/M tallies.Inferred from source code, not pixels.
frame-unclear / interaction-unclearThe captured evidence is insufficient to decide and the shader doesn't settle it either. Honest "can't tell".Neither pixels nor code suffice.
n/a (family level)The whole probe family doesn't apply — e.g. music probes on a silent piece, dual-input probes on a cursor-only piece. Families are gated by what meta.yaml declares.Structural.

Dimensions (the 1–5 numbers)

Dimensions are NOT computed from probes. They are a second, independent judgment axis: six qualities each scored 1–5 directly against written anchors in taste.md §Dimensions. Only the 1, 3, and 5 anchors are defined; 2 and 4 are "between"; ambiguity rounds down. Example, Motion: 1 = "one-scale pulse locked to audio, flat between beats" … 5 = "always mixing at multiple scales".

The six: palette_cohesion, composition, motion, intensity (dynamic range), depth, form_ending (n/a for unbounded pieces). The "avg dims" number in the studio UI is a display-only mean of the numeric dimensions (defined in studio/server.mjs compositeOf(), not in the rubric).

The claim check (binary gate)

Before any scoring: pull the thesis from meta.yaml, restate it, and ask "do the frames deliver it?" Pass/fail, no scale. A failed claim is automatically the top fix — "Fix the lie first. Polish second."
