Lesson 0001 — The pipeline map

~5 minutes · sources: skills/vjay-new-piece/SKILL.md, skills/vjay-iterate/SKILL.md · glossary: grade vocabulary

The win: after this lesson you can place any artifact in the repo (a critique file, an inspect frame, a lint, a verdict chip) at its exact step in the creation pipeline — so when we dissect the critic in lesson 0002, you'll know precisely whose hands it receives work from and whose hands receive its output.

The flow, end to end

A piece is born through /vjay-new-piece (or /vjay-from-url for the download+scaffold part). The steps that matter:

1  BRIEF GATES      thesis, canonical algorithm ref, eye-landing candidates,
                    warm-palette plan, idle behaviour, architecture rationale
                    → written into meta.yaml `notes:` — this becomes "the claim"
2  AUDIO            download → bin/analyze-audio.mjs --stems
                    → audio.analysis.json (bpm, beats, sections, stems)
3  BUILD            shader.frag / sim.frag / layers + meta.yaml
4  RENDER EVIDENCE  bin/inspect-music.mjs 
                    → 6 section-anchored stills + 4-5 multi-window clips
5  MACHINE LINTS    lint-palette / lint-idle / lint-composition / audit-piece
                    → hard blockers, fixed before any critic sees the piece
6  SELF-CRITIQUE    the building session writes critiques/-v1.md
                    (cold-open read; NOT the independent critic)
7  ITERATE LOOP     /vjay-iterate — the artist & critic loop (below)
8  COMMIT BUNDLE    piece + brainstorm + critiques + evidence, one commit

Two design choices worth noticing. First, the claim is written before the code (step 1) — the critic later grades the piece against its own stated thesis, not against a generic ideal. Second, machine lints run before the critic (step 5) — cheap deterministic checks (cool-pixel percentage, frozen idle, Y-split composition) so the expensive judgment-based critic never wastes a pass on mechanical failures.

The artist & critic loop

Inside /vjay-iterate there are two roles with deliberately unequal powers:

The artist is the main session — it can Edit shaders. The critic is a separate spawned agent with read-only tools (it can view images and read code but cannot edit). This asymmetry is the point: the murmuration stress test showed a builder grading its own work scores it 5–7 composite points too generously, and a critic that can edit drifts into applying its own fixes instead of judging.

            ┌────────────────────────────────────────────┐
            ▼                                            │
   render evidence (stills + clips)                      │
            │                                            │
   CRITIC reads: taste.md, frames, clips, shaders,       │
   meta.yaml claim, prior critiques                      │
            │                                            │
   writes critiques/-vN.md  (prose + YAML tail)    │
            │                                            │
      verdict?                                           │
      ├── chef-doeuvre / ship-it  → DONE (commit)        │
      ├── needs-tweak → ARTIST applies top_fix ──────────┘   (≤ 8 laps)
      └── structural-rethink / premise-wrong → STOP, hand to Louis

The two STOP verdicts exist because of a measured failure mode: forcing iteration past a structural-rethink verdict gains at most ~4.5 composite points and plateaus by iteration 4–5. Parameter-tuning cannot rescue a piece whose architecture is wrong — that decision belongs to Louis at the brief level.

What each artifact is for

critiques/<slug>-vN.md is simultaneously: (a) the loop's control signal (the YAML tail is parsed to route the next lap), (b) the permanent record the studio grades view renders (chips, panels, v/Shift+V), and (c) the input to the next critique (the critic reads its predecessors to avoid oscillating fixes). evidence/<slug>-vN/ freezes the exact frames graded — inspect frames get overwritten by later runs, evidence doesn't.

Check yourself — why does the critic run as a separate read-only agent, rather than the building session grading its own work?
The separation is a bias guard, not a performance choice: independent eyes (anchored only in taste.md) plus no ability to touch the code. Source: vjay-new-piece SKILL.md §11 and vjay-iterate SKILL.md §critic-agent notes.
