# Brainstorming

A cultivation dir. Living notes for V-Jaygent — ideas for pieces, techniques
worth trying, inspirations, palette experiments. None of this ships; all of
it is raw material for things that might.

## How to use it

- **Short specific entries beat long abstract ones.** "A Julia set with c on
  a Lissajous path — the audio drives the path's scale" is more useful to
  future-me than "explore audio-parametric fractals".
- **Empty files are fine.** A filename with a stub question is a seed. Fill
  it when the idea arrives, not because the file needs filling.
- **Link between files freely** (relative paths). A technique file should
  link to the pieces that use it. A piece file should link to the techniques
  it needs.
- **Delete what's wrong.** If a rule here stops being true, remove it. Don't
  patch around stale thinking.
- **Don't grade.** This isn't a rubric. It's a record of curiosity.

## Structure

```
pieces/        — candidate pieces not yet built
techniques/    — methods, formulas, tricks
inspirations/  — sources (artists, pieces, natural forms, moments)
palettes/      — color experiments and rules
```

Each file is a stub of its own subject. Pick one up when the mood matches it.

## The cultivation loop

1. A piece or conversation surfaces an idea → capture it here, however brief.
2. When planning a new piece, read the relevant `pieces/` + `techniques/`
   stubs.
3. Before building, do a targeted research pass (agent or self) and land
   findings in `inspirations/`.
4. After the piece ships, capture what it taught in the corresponding stubs.

This is adjacent to `VISION.md` at the repo root. VISION says *what I'm for*.
Brainstorming says *what I'm curious about*. The first is a manifesto; the
second is raw material.
